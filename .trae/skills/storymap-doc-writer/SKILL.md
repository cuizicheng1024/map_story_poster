# StoryMap Doc Writer

## 适用场景（何时调用）

- 用户要为 StoryMap 项目撰写/改写参赛材料（案例稿、申报表、路演稿、摘要、亮点、问答）。
- 用户给出主办方模板/栏目（如“应用场景、技术方案、产业化可行性、市场前景、融资情况”）并要求填充。
- 用户要求输出指定长度（如 300 字摘要、1 万字案例稿）或指定格式（Markdown/Word/PPT 大纲）。
- 用户要求把内容“对齐 README/仓库现状”，或要求“更强调 AI+时空数据创新应用 / LBS 数据价值”。

## 输出原则

- 以项目仓库实际功能为准：主页/人物页/未收录人物自动生成/任务进度/服务端模型代理/地名解析与缓存等。
- 语言默认中文；如用户指定英文或双语则切换。
- 结构优先对齐对方模板：严格按栏目顺序、标题层级与字数约束输出。
- 避免泄露密钥与敏感信息：不得引用或复述 .env 中的任何 key。
- 需要“可投递”的完成度：包含封面信息占位（作者/单位/联系方式）、投稿邮箱/主题格式等可直接替换项。

## 工作流程（建议）

1. 读取用户提供的征集/申报要求与限制（字数、格式、栏目、目标方向）。
2. 从仓库 README 与现有材料中抽取“项目定位、目标用户、核心流程、亮点、成熟度、落地价值”。
3. 组织成一份可投递文稿：
   - 摘要与关键词
   - 背景与痛点
   - 技术方案与创新点（架构图可用 ASCII/文本描述）
   - 数据与治理（地名消歧、缓存、离线索引、质量校验）
   - 产业化可行性（产品形态、成本结构、商业模式、迭代路线、里程碑）
   - 市场前景与增长路径
   - 融资情况（如未知，使用“未融资/计划融资”中性表述，不编造）
   - 附件建议（演示链接、截图清单、试点反馈模板）
4. 根据用户反馈进行定稿：更贴近某一赛道（AI+时空/LBS/公共服务/交通/城市治理等）或更强调商业化。

## 常用模板片段

### 300 字以内项目简介（背景/创新/场景）

要求：一句话定位 + 目标用户 + 解决痛点 + 关键创新点 + 典型应用场景。

### “六栏申报表”填充

1) 应用场景描述（项目背景、目标用户、痛点解决）  
2) 技术方案与创新性（技术架构、关键创新点、成熟度、技术自主性、知识产权）  
3) 产业化可行性（产品形态、成本结构、商业模式、迭代路线、成果、产业价值）  
4) 市场前景（市场空间、差异化优势、增长路径）  
5) 融资情况（已融资/计划融资/现金流预期）  
6) 其他补充说明（附件清单）  

## 需要用户提供的最小信息（若缺失则用占位）

- 作者/单位/联系方式（用于封面与投稿邮件）
- 目标赛道/方向（如“AI+时空数据创新应用”）
- 是否已有试点/用户反馈/数据指标（若没有，不虚构）
